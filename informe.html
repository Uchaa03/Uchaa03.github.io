Modelos de ejecución Cliente-Servidor

La arquitectura cliente servidor es de las mas utilizadas en la construcción de aplicaciones en la actualidad, esta se basa una estructura principal en la que trabajan dos componentes un cliente que utiliza una serie de servicios comunicandose con el servidor que los proporciona

Arquitectura de dos Capas

Esta es la arquitectura tradicional conocida como arquitecturas de dos capas/niveles, un cliente utiliza una interfaz de trabajo por el que realiza las peticiones al servidor y el servidor las responde.

  • El desarrollo en dos capas es funcional pero no es lo más eficiente para aplicaciones complejas
  • Es una arquitectura fácil de emplear, pero no funcionan bien con gestiones grandes de datos
  • Requiere mucho control de versiones a la hora de realizar cambios ya que la mayoría de la aplicaicón lógica la lleva la interfaz de cliente que pide las solicitudes
  • La seguridad es compleja debido a la alta demanda y acceso a la base de datos
  • Dificultades en cuanto implemnetaciones y esclabilidades futuras
Arquitectura de tres Capas

Esta a diferencia de la otra es mas organizada debido a que la divide en tre secciones la capa lógica la física y la de la aplicación

  • Esta tiene mejor desarrollo ya que se pueden hacer las tres capas a la vez por distintios desarrolladores
  • Mayor escalabilidad, y rendimiento
  • Tiene mayores niveles de seguridad debido a que las capas no se comunican directamente

Lenguajes de Programación Web

JAVASCRIPT

Es uno de los lenguajes mas usados en la actulidad, permite la creación de páginas dinámicas y aplicaciones web interactivas, es interactivo, compatible con todos los navegadores modernos facil de aprender y usar, basado en envetos, asíncrono muy util para la carga de datos de una petición de servidor y tiene una gran cantidad de frameworks y librerias para cumplimentarla

TYPESCRIPT

TypeScript va mas allá de ECMASCRIPT haciendo de JS una versión mejorada con un versión de este con tipados para la programación orientada a objetos, este lee perfectamente el JS per a la inversa hay que hacer una conversión, además de tener soporte directo para módulos e interfaces y decoradores

PYTHON

Es otro legnuaje fácil de enteder reconocido gracias ello de tipo scripting, es fácil y sencillo a la hora de usar y leer este es un lenguaje interpretado y orientado a la programación de Objetos (POO), tiene una biblioteca interna muy amplia, y es multiplatadorma al igual que JS

PHP

Es un lenguaje de progrmación de script de servidor que se utiliza para el deserrallo de sitios y aplicaciones web, muy útil para la interacción con bases de datos y formularios, además de usarse en sistemas de gestión de contenidos como Wordpress Drupal o Joomla

Ruby

Es un lenguaje orientado a objetos, también conocido com Rails, sigue un modelo MVC de vista controlador útil para el desarrollo y mantenimiento al separar datos de lógica e interfaz de usuario, esta diseñado para la productividad y la construcción de aplicaciones rápida y eficientes
